GIT REBASE vs GIT MERGE

Made a file and added the file learn.txt

Verifying using :

git log

initially we have no branch except master

git branch
This command shows how many branches are present and on which branch we are present.

git branch shows the branch on which we are present which is shown by branch name preceded by asterisk.

made feature branch

git branch branch_name

switch from one branch to another

git checkout branch_name

checkout command will take us to the branch whose name we entered in the “git checkout branch_name” command.

adding first commit on feature branch

updating master branch with second commit

See status by:

git log

updating feature branch with second commit

And confirming commit status by git log

updating feature branch further

So as of now we are in the following condition:

Master branch has 2 commits:
second commit
added lessgo.txt

feature branch is updated as:
fourth commit
third commit
second commit

updated master branch with “M3 master 3rd commit”

Now the most recent commit in master branch is M3 master 3rd commit and feature branch has fourth commit and now we want the commmit history as:

fourth commit
M3 master 3rd commit
second commit
added lessgo.txt

To get the above status and Combining the recent commits and for a clear history of commit we use:

git rebase branch_name

I got an error as the first commit on feature branch was not merged so we need to remove the first commit on feature branch and as the feature branch was based on second commit of master branch so the first commit on feature is needed to be removed and we will get the desired output.
